48// Un arbol B se define dado un grado M
49// Cada nodo tiene ahora una lista de valores, no como en el arbolAVL que este tiene solo un valor
50// La cardinalidad de esta lista se le denomina "claves", si el el grado es M=5, cada nodo tiene a lo mucho 4 claves
51
52// Cada nodo (excepto la raiz y hojas) puede tener entre M/2 y M hijos
53// Cada nodo (excepto la raiz) puede tener entre M/2-1 y M-1 claves
54// La raiz puede tener entre 2 y M hijos
55// Si un nodo tiene exactamente k claves, entonces esta tiene k+1 hijos
56
57// Ejemplo: Un arbol B con grado M=5:
58// - Cada nodo tiene entre 3 y 5 hijos, y entre 2 y 4 claves
59// - la raiz puede tener entre 1 y 4 claves
60
61// La lista de un nodo esta ordenada: (k_1 < k_2 < ... < k_{M-1})
62// Entre cada una de esas claves, hay un hijo -> |k_1|k_2|...|k_{M-1}| en donde el simbolo '|' denota que ahi hay un puntero a hijo
63// Aqui el chiste es que todos los valores que se almacenan en los hijos sigue estando ordenado:
64// es decir, todas las claves del hijo que esta por ejemplo en |k_1 son menores que la clave k_1
65// todas las claves que estan entre k_1|k_2 estan en el rango de k_1 < hijo < k_2 y asi sucesivamente...
66
67// Todas las hojas estan al mismo nivel
68
69// === Estructura de un nodo ===
70// T *claves -> [0, 1, ..., Mgrado-1]: arreglo dinamico con a lo mucho Mgrado-1 claves (uno mas para detectar overflow)
71// Nodo **hijos -> [0, 1, ..., Mgrado]: arreglo dinamico de punteros a hijos (uno mas para detectar overflow)
72// numClaves: cantidad de claves actuales en el nodo
73// esHoja: indicador para saber si es hoja
74
75// La operacion de dividir el arbol se ejecuta cuando se intenta tener mas claves de las permitidas
76// (Mgrado claves cuando a lo mucho se puede Mgrado-1). La clave del medio (indice Mgrado/2) sube al padre.
77
